Project Brief: Daily AI Artist (News → Abstract Art → POD + Social)
1) Goal

Build a fully automated daily pipeline that:

pulls “news of the day” from selected sources

uses OpenAI to extract 5–10 abstract “creative directions”

generates one unique abstract artwork in a consistent house style (via Stable Diffusion, customised)

stores everything (inputs, prompts, outputs, decisions, performance signals) in a lightweight DB

publishes the daily drop (image + caption) to selected channels (initially: a simple web page + optional social autopost)

Key constraint: minimal human time day-to-day. Human only intervenes when quality/safety checks fail or when you explicitly “approve mode”.

2) Non-goals (to prevent scope creep)

No full e-commerce site (use POD marketplaces / links).

No manual graphic design workflow (the point is consistent AI style).

No complex ML training pipeline on day 1 (start with prompt + reference approach; add LoRA/fine-tune later).

3) End-to-end Workflow (Daily Run)

Step A — Collect (News Ingestion)

Pull daily articles/headlines via:

RSS feeds (fast, reliable)

optional: curated sources list (URLs)

Normalize into a single JSON payload: {source, title, snippet, url, published_at}

Step B — Analyze (Theme Extraction via OpenAI)

Prompt OpenAI to output:

a ranked list of 10 candidate themes (abstracted)

a risk screen (no hate/violence glorification, no personal data, no direct political persuasion copy)

for each theme: suggested “visual tokens” (shapes, texture, composition, mood) and banned elements (text, flags, weapons, faces, etc.)

Step C — Select (Decision Policy)

Default mode: automatically pick top 1 theme based on scoring rules.

Optional mode: “Human approval” where you choose 1 of 10 in a UI.

Step D — Generate Art (Stable Diffusion)

Generate 1–4 candidates using the chosen theme.

Apply strict style constraints:

no text, no logos, no recognisable people, no gore

restrict colour palette (list of allowed colours / exclude list)

enforce composition motifs (lines/dots/blocks etc.)

Select best candidate automatically using simple heuristics (see Quality Gates).

Step E — Persist (Database Logging)
Store:

raw news items

analysis output (themes, scores)

chosen theme

generation parameters (seed, model, negative prompt, control settings)

final image path + hash

caption copy + hashtags

publish status + links

later: performance signals (likes, clicks, sales)

Step F — Publish

Publish image + caption to:

local web page (MVP)

optional: Instagram posting automation (later, via Meta API or 3rd party tool)

4) “Agents” and Responsibilities (so it’s modular)

Think of each “agent” as a small module with one job:

Ingestion Agent

fetch RSS/articles

dedupe + clean

Analysis Agent (OpenAI)

produce themes + art directions + safety flags

Curator Agent (Policy + Ranking)

scores themes and picks one

can switch between “auto” and “approve”

Artist Agent (Stable Diffusion)

generates candidates with house style constraints

outputs image(s) + metadata

Quality Gate Agent

rejects outputs that violate rules (text detected, wrong palette, unsafe content)

if rejected: regenerate with altered seed/prompt

Publisher Agent

generates caption (OpenAI) in consistent “artist voice”

posts to web + (later) social + stores final URLs

Analytics Agent (later)

pulls engagement/sales signals

feeds back into ranking prompts

5) Tech Stack (Replit-friendly)

Language: Python 3.11+
Runtime: Replit (scheduled runs or manual run button)

Core libs

feedparser (RSS)

requests

python-dotenv

sqlite3 (built-in) or tinydb

openai (OpenAI API client)

Image processing: Pillow

Optional quality checks:

pytesseract or a lightweight OCR alternative (for text detection)

simple palette check via Pillow

Image Generation

Option A (MVP, easiest): call a hosted Stable Diffusion API (e.g., Replicate / etc.)

Option B (more control): run SD via a service endpoint you host elsewhere

Option C (best style consistency): SD + LoRA (trained later), still served via API

Storage

SQLite DB file: data/app.db

Images on disk: outputs/YYYY-MM-DD/

UI

MVP: simple Flask page showing the latest drop + archive.

Optional: a “approve today’s theme” page.

7) House Style System (Consistency without daily human work)

Implement style through a style config:

style_config.json

palette: allowed/banned colours

composition motifs: e.g. “lines + dots, asymmetry, high negative space”

forbidden content: text, faces, flags, weapons, maps, identifiable symbols

stable diffusion settings defaults:

resolution for printing (e.g. 4500×5400 for tees, depending on POD specs)

CFG scale range

steps

sampler

negative prompt

reference strategy:

optional “style reference embeddings / LoRA name” (when you introduce it)

optional “reference image influence strength” (if supported)

Important legal/ethical constraint: don’t “clone” a living artist’s style so closely it’s recognisable. “Influence” and “constraints” are safer than direct imitation.

8) Quality Gates (Human-in-the-loop without daily effort)

Automatic checks before publishing:

Text detection: reject if OCR finds characters above threshold

Palette check: reject if too much banned colour

Safety check: OpenAI text safety pass on caption + theme; basic keyword heuristics on prompts

Uniqueness check: hash similarity vs last N days (reject if too similar)

Fallback policy:

regenerate up to N times with modified seed / slightly altered prompt

if still failing: switch to “safe abstract fallback prompt” and publish that

Human intervention points (optional):

approve theme

approve final candidate only if quality gates fail repeatedly

10) MVP Milestones (pragmatic)

Day 1: RSS ingestion → OpenAI themes → print to console + save JSON

Day 2: Stable Diffusion API generation → save image files

Day 3: SQLite logging + archive page

Day 4: Quality gates + regeneration loop

Day 5: Publisher (web) + daily scheduler

Later: Social autopost + POD API integration + analytics feedback loop

11) Known Compromises (log these as a refactoring backlog)

Using hosted SD API instead of fully controlled model pipeline (faster, less control).

Prompt-based style consistency before LoRA/fine-tune (acceptable MVP, weaker “signature style”).

Basic quality heuristics vs robust computer vision checks (good enough early, may misclassify).

SQLite vs scalable DB (fine early; upgrade if volume grows).
